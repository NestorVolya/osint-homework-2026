# ArkhamMirror (SHATTERED) — Платформа для аналізу документів та розслідувань

## Що це?

**ArkhamMirror** — це локальна, AI-вбудована платформа для глибокого аналізу документів та оперативних розслідувань. Вона розроблена для журналістів, розслідувачів, адвокатів та OSINT-фахівців, які працюють з великими обсягами документів.

На відміну від хмарних рішень, ArkhamMirror працює локально на вашій машині — ваші дані ніколи не передаються на зовнішні сервери.

## Принцип роботи

ArkhamMirror організовує роботу навколо єдиного циклу розслідування:

```
┌──────────┐    ┌──────────┐    ┌──────────┐    ┌──────────┐    ┌──────┐
│          │    │          │    │          │    │          │    │      │
│  INGEST  │───▶│ EXTRACT  │───▶│ ORGANIZE │───▶│ ANALYZE  │───▶│ ACT  │
│          │    │          │    │          │    │          │    │      │
│ PDF/DOCX │    │ OCR/NER  │    │  Graph   │    │ Anomaly  │    │Export│
│ IMG/TXT  │    │ Claims   │    │ Timeline │    │ ACH/ACT  │    │Report│
│ CSV/JSON │    │ Relations│    │ Vectors  │    │ Patterns │    │ FOIA │
└──────────┘    └──────────┘    └──────────┘    └──────────┘    └──────┘
```

### 1. INGEST — Прийом документів

Завантажуйте документи в будь-яких форматах:
- **Текст:** TXT, MD, HTML, DOCX, PDF
- **Зображення:** PNG, JPG, GIF, WebP
- **Метаформати:** CSV, JSON

Система автоматично виявляє дублікати, нормалізує метадані, ставить у чергу на обробку, підтримує пакетне завантаження.

### 2. EXTRACT — Вилучення даних

```
Документ
   │
   ├── OCR ──────────── PaddleOCR (стандарт)
   │                    Vision LLM (складні макети)
   │
   ├── NER ──────────── PERSON / ORG / GPE / DATE / ...
   │
   ├── Claims ────────── фактичні твердження → прив'язка до джерел
   │
   └── Relations ─────── зв'язки між сутностями
                         8 стратегій розбиття тексту
```

### 3. ORGANIZE — Організація інформації

**Граф-аналіз (10+ режимів):**

```
Force-Directed   Hierarchical   Geographic   Causal
     ●─●              ●            🗺️          A→B
    ╱│╲            ╱─┴─╲                      ↓
   ● ● ●          ●     ●                     C
```

| Режим | Призначення |
| --- | --- |
| Force-Directed | Інтерактивна мережева топологія |
| Hierarchical | Деревовидна структура |
| Geographic | Геопросторове розташування |
| Causal | Причинно-наслідкові зв'язки |
| Temporal | Часова еволюція мережі |
| Link Analysis | Стиль i2 Analyst Notebook |

**Часова шкала:**

```
[2020]──────[2021]──────[2022]────────────────[2023]
   │            │           │                    │
  подія A     подія B   КОНФЛІКТ ⚠️           подія D
                        (суперечність дат)
```

**ACH Matrix:**

```
             │ Гіпотеза 1 │ Гіпотеза 2 │ Гіпотеза 3
─────────────┼────────────┼────────────┼────────────
Доказ A      │     C      │     I      │     N/A
Доказ B      │     I      │     C      │     C
Доказ C      │     N/A    │     I      │     I
─────────────┼────────────┼────────────┼────────────
Score        │    -1      │    -2      │    +1  ✓
```

*C = consistent, I = inconsistent*

### 4. ANALYZE — Аналіз

```
Дані
 │
 ├── Аномалії ─────── статистичні / контекстові / колективні
 │                    └── LLM-інтерпретація
 │
 ├── Суперечності ─── конфліктуючі твердження між документами
 │                    └── оцінка важкості + відстеження резолюції
 │
 ├── Паттерни ──────── повторювальні / поведінкові / часові
 │                    └── кореляційний аналіз
 │
 ├── Достовірність ─── MOM / POP / MOSES / EVE чеклисти
 │
 └── Медіафорензика ── EXIF → ELA → C2PA → зворотний пошук
```

### 5. ACT — Дія

```
Результати
 │
 ├── Експорт: JSON / CSV / PDF / DOCX
 ├── Звіти: резюме / профілі сутностей / ACH / таймлайн
 ├── Листи: FOIA-запити / скарги / юридичні листи
 └── Пакети: повний знімок розслідування + версіонування
```

## Ключові можливості

### 🤖 AI Junior Analyst

Вбудований помічник LLM: автоматичне виявлення аномалій, пошук суперечностей, розпізнавання паттернів, оцінка достовірності, синтез інсайтів.

### 🔍 Гібридний пошук

```
Запит користувача
       │
  ┌────┴────┐
  │         │
Semantic  Keyword
(embeddings) (full-text)
  │         │
  └────┬────┘
       │
   Hybrid ← налаштування ваги
       │
   Результати + фільтри
```

### 🌐 Граф-метрики мережі

- Централізованість: degree / betweenness / closeness
- Виявлення спільнот
- Пошук шляхів: найкоротший / усі / критичні
- Виявлення циклів

### 📷 Медіафорензика

- EXIF дані → GPS, камера, час
- Error Level Analysis (ELA) → виявлення монтажу
- Перцептивне гешування → пошук варіантів зображення
- C2PA верифікація → цифровий підпис контенту

### 🔐 Приватність та безпека

- 100% локальна робота (air-gap розгортання)
- Без телеметрії
- Ролі: Admin / Analyst / Viewer
- HTTPS через Traefik + Let's Encrypt

## Інтеграція з MindsDB (контекст ДЗ-03)

ArkhamMirror зберігає всі метадані документів у PostgreSQL — це дозволяє підключити MindsDB напряму і виконувати AI-запити поверх завантажених матеріалів.

```
┌─────────────────┐        ┌─────────────────┐
│   ArkhamMirror  │        │     MindsDB     │
│                 │        │                 │
│  Documents ─────┼──────▶ │  arkham_db      │
│  Entities       │ pgSQL  │  (PostgreSQL    │
│  Relations      │  5432  │   integration)  │
│  Metadata       │        │                 │
└─────────────────┘        │  qwen_osint ────┼──▶ Ollama
                           │  (ML Engine)    │    qwen2.5:3b
                           └─────────────────┘
```

```sql
-- Крок 1: Підключити базу Arkham до MindsDB
CREATE DATABASE arkham_db WITH ENGINE = 'postgres'
PARAMETERS = {
  "host": "host.docker.internal",
  "port": 5432,
  "user": "arkham",
  "password": "***",
  "database": "arkhamdb"
};

-- Крок 2: Переглянути таблиці Arkham
SHOW TABLES FROM arkham_db;

-- Крок 3: AI-агент аналізує документи з Arkham
SELECT a.filename, a.entity_name, m.answer
FROM arkham_db.document_entities AS a
JOIN mindsdb.qwen_osint AS m
WHERE m.question = CONCAT('Analyze OSINT risk for entity: ', a.entity_name)
LIMIT 10;
```

cd ArkhamMirror
cp .env.example .env
