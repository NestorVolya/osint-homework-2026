## 1. Виявлення спільноти в графах

**Спільнота (community)** — група вузлів, де внутрішні зв'язки щільніші за зовнішні.
У патентному графі це означає: патенти у спільноті схожіші між собою, ніж із патентами з інших кластерів.

**Чому це важливо для OSINT:**
Виявлення спільноти дозволяє побачити **приховану структуру** даних без попередніх гіпотез.
У патентному ландшафті кластери можуть відповідати технологічним підсистемам,
школам розробки або хронологічним епохам — але це завжди **гіпотеза, яка потребує верифікації**.

## 2. Алгоритм Лувена (Louvain)

**Автори:** Blondel, Guillaume, Lambiotte, Lefebvre (2008)  
**Джерело:** *Fast unfolding of communities in large networks*, JSTAT 2008 (10), P1000

**Принцип роботи:**

```
Крок 1 (Phase 1 — Greedy):
  Для кожного вузла: перевірити, чи збільшить переміщення до сусідньої спільноти Q
  Повторювати до стабілізації

Крок 2 (Phase 2 — Aggregation):
  Кожна знайдена спільнота → стискається у мета-вузол
  Граф перебудовується

Повторювати обидва кроки ієрархічно → до відсутності покращень Q
```

**Параметри (з Gephi):**
- Randomize: On — кращий результат, але не детерміністичний
- Use edge weights: On — враховує вагу ребер (у нашому датасеті всі ваги = 1)
- Resolution: 1.0 — стандартне значення; < 1.0 = менше великих кластерів; > 1.0 = більше малих
- Seed: 0 (початкова точка для randomize)

**Чому Python (networkx) і Gephi дають різні ID:**
Обидва реалізують той самий алгоритм, але з різними seed.
Результат: інші числові ID кластерів, але **структурно ідентичні** спільноти.
Підтвердження: топ-кластер Python = 331 вузол, Gephi = ~240 вузлів (3.05% × 7869).

## 3. Модульність Q — формула та інтерпретація

### Формула

```
Q = (1/2m) × Σᵢⱼ [Aᵢⱼ - (kᵢ·kⱼ)/(2m)] × δ(cᵢ, cⱼ)
```

де:
- `m` = кількість ребер (8819)
- `Aᵢⱼ` = 1 якщо ребро між i і j існує, 0 інакше
- `kᵢ` = ступінь вузла i (кількість ребер)
- `(kᵢ·kⱼ)/(2m)` = **очікувана** кількість ребер між i і j у випадковому графі
- `δ(cᵢ, cⱼ)` = 1 якщо вузли в одній спільноті, 0 інакше

**Спрощено:** Q = (реальні зв'язки всередині кластерів) − (очікувані зв'язки у випадковому графі)

### Шкала

| Q | Інтерпретація |
|---|---------------|
| < 0.0 | Гірше за випадковий граф |
| 0.0 – 0.3 | Слабка кластерна структура |
| 0.3 – 0.7 | Помірна структура |
| 0.7 – 0.9 | Сильна структура |
| **> 0.9** | **Відмінна структура** |

### Чому **0.956** відмінно

Значення 0.956 означає, що патентний граф має **дуже виражену природну кластерну структуру**.
Це не дивно: датасет побудований на **семантичній схожості** патентів (Patent similarity search).
Патенти в одній технологічній ніші семантично близькі між собою і далекі від інших ніш.

**Порівняння з типовими графами:**
- Соціальна мережа (Facebook): Q ≈ 0.3–0.5
- Цитатна мережа наукових статей: Q ≈ 0.6–0.8
- **Патентний similarity граф (цей датасет): Q = 0.956**

## 4. Modularity Class

**Що це:** ціле число (0..701), присвоєне кожному вузлу після запуску Modularity в Gephi.

**Важливо розуміти:**
- Клас 0 ≠ "найкращий" або "найбільший" — це просто ідентифікатор
- Клас 112 є найбільшим (3.05% вузлів) лише тому, що Gephi пронумерував його так
- Той самий граф при повторному запуску дасть **інші числові ID** (якщо Randomize: On)

**Практичне використання:**
1. Appearance → Partition → Modularity Class → колір кожного класу унікальний
2. Data Laboratory → Export → колонка `Modularity Class` у CSV
3. Фільтрація: Filters → Partition → Modularity Class → обрати один клас для аналізу

**Зв'язок з атрибутом `community` у GEXF:**
У нашому GEXF вузли мають атрибут `community` (від Python Louvain) та `Modularity Class` (від Gephi).
Це різні числа для тих самих кластерів — відображення різних seed.

## 5. Інші метрики графа

### Базові характеристики (з Gephi Контекст)

| Метрика | Значення |
|---------|---------|
| Вузли | 7869 |
| Краї | 8819 |
| Тип | Неорієнтований граф |
| Середній ступінь | 2 × 8819 / 7869 ≈ **2.24** |
| Щільність | 2 × 8819 / (7869 × 7868) ≈ **0.000285** |

### Середній ступінь (Average Degree)

**Середній ступінь ≈ 2.24** означає: кожен патент у середньому схожий на 2-3 інших.
Для патентного similarity-графа це **очікувано низьке значення** — граф розріджений.
Причина: більшість патентів мають вузьку специфіку → мало близьких сусідів.

### Щільність графа (Graph Density)

**Щільність ≈ 0.000285** (~0.03%) означає: реалізовано 0.03% від усіх можливих ребер.
Це **дуже розріджений граф** — типово для патентних і цитатних мереж.
Висока щільність у таких графах підозріла: означала б штучні зв'язки.

### Розподіл розмірів кластерів (Size Distribution)

З Modularity Report (графік):
- Більшість із 702 кластерів малі (1–10 вузлів) — горизонтальна смуга внизу
- 10–15 великих кластерів (100–240 вузлів) — виступи вгору
- Максимум: ~240 вузлів (Клас 112, 3.05%)

Це **степеневий розподіл** — характерний для реальних мереж (law-tail distribution).

## 7. Значення для OSINT-аналізу

### Що модульність каже аналітику

| Висока Q (0.956) означає | Не означає |
|--------------------------|------------|
| Природна кластерна структура в даних | Що кластери відповідають реальним організаціям |
| Алгоритм знайшов стабільні групи | Що зв'язки всередині кластера причинно-наслідкові |
| Граф придатний для подальшого аналізу | Що хтось свідомо створив ці групи |

### Межі методу

1. **Роздільна здатність (Resolution limit):** Louvain не може виявити кластери менші за √(2m) ≈ √17638 ≈ 133 вузлів.
   Мікро-кластери (3–10 вузлів) теоретично можуть бути "поглинуті" більшими.

2. **Нестабільність:** Різні seed → різні ID (але подібна структура).
   Перевірка стабільності: запустити 5 разів, порівняти розподіл розмірів.

3. **Semantic != Organizational:** Патенти в одному кластері схожі за **текстом назви**,
   але не обов'язково за **організаційним походженням**.
   Правильна фраза: "структурно схоже на", а не "ці патенти від одного НДІ".
