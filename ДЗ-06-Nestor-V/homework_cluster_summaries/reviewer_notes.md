## Загальна оцінка

AI-чернетка містила **3 категорії проблем**:
1. Впевнені формулювання там, де потрібні гіпотези
2. Домислені зв'язки з організаціями ОПК без джерел
3. Ігнорування суперечливих тем у кластерах

Після ручної правки збережено ~60% тексту чернетки.

## Правки по кластерах

### Cluster 496 (глісери та пропульсія)

**Вилучено:**
- ~~"Ці патенти свідчать про розробку пропульсивних систем для БЕК"~~ → немає джерела
- ~~"Крыловський ДНЦ активно патентував у цій сфері"~~ → з назв патентів не випливає

**Виправлено:**
- "розробка рушіїв для БЕК" → "структурно схоже на технологічну базу малих надводних платформ"

**Залишено з поміткою [ГІПОТЕЗА]:**
- Зв'язок між кластерами 496 і 285 (пропульсія + навігація) — цікавий патерн,
  але потребує перевірки по патентоволодільцях

### Cluster 359 (бойові та спеціалізовані кораблі)

**Вилучено:**
- ~~"Це ядро воєнно-морського R&D РФ"~~ → надмірне узагальнення

**Виправлено:**
- "B63G (бойові кораблі)" → додав як питання для перевірки, а не як факт

**Додано рев'юером:**
- RU2834069C1 (2025) — потребує окремої перевірки через свіжість

### Cluster 234 (рятувальні системи та імпеллери)

**Вилучено:**
- ~~"Імпеллерні судна — очевидна подвійного призначення технологія"~~ → "очевидна" неприйнятно
- ~~"Три патенти RU2718xxx подані тим самим НДІ"~~ → фантазія, не підтверджено

**Виправлено:**
- "Серія RU2718xxx" зазначена як питання для перевірки, а не факт

**Залишено:**
- Опис підгруп (рятувальні + імпеллерні) — структурно вірно, підтверджено назвами

---

### Cluster 285 (навігація та автоматичне управління)

**Вилучено:**
- ~~"Цей кластер — R&D для автономних суден ВМФ РФ"~~ → пряма галюцинація
- ~~"Патенти після 2022 відображають воєнну потребу"~~ → кореляція ≠ причина

**Виправлено:**
- "воєнний контекст" → "може свідчити про R&D у сфері автономного судноводіння"
- Додано: "На підставі лише графа не можна стверджувати зв'язок із безпілотними системами"

---

### Cluster 163 (маломірні судна)

**Вилучено:**
- ~~"Приватні винахідники можуть бути прикриттям для держзамовлення"~~ → конспірологія без підстав
- ~~"Тримаран — стандартна платформа для безпілотних катерів типу БЕК"~~ → неперевірено

**Виправлено:**
- "стандартна платформа" → "структурно схожі на базові елементи дизайну"

**Залишено:**
- Опис авторських серій (Поль, Мошконов) — це реальний паттерн, підтверджений назвами
- Питання про тримаранну конструкцію — доречне для перевірки

## Системні проблеми LLM

### 1. Overconfidence у причинно-наслідкових зв'язках

LLM систематично перетворювала структурні патерни графа на причинно-наслідкові
висновки. Приклад:
- Граф: патенти А і Б в одному кластері
- LLM: "А і Б розроблялися в межах однієї програми"
- Правильно: "А і Б структурно схожі, можливо — одна область R&D"

### 2. Ігнорування нерелевантних тем у кластерах

Кластери містять несподівані теми (настільні ігри, медицина, хімія серед морських патентів).
LLM їх або ігнорувала, або виключала як "очевидно нерелевантні".
Правильний підхід: відзначити аномалії, не прибирати їх з аналізу.

### 3. Надмірна конкретність гіпотетичних організацій

LLM кілька разів назвала "Крыловський ДНЦ" та "ФГУП" без джерел.
Навіть у режимі гіпотез — конкретні назви організацій без підтвердження
перетворюються на псевдофакти після кількох ітерацій.

## Що залишилось без перевірки (ризики)

- Патентоволодільці кластерів — не перевірені через відсутність даних у Excel
- Рік подачі патентів vs рік реєстрації — не аналізувався
- MPK-коди у деталях — потребують звірки з офіційним класифікатором
- RU2834069C1 (2025, сторожовий корабель) — не перевірений за зовнішнім джерелом

## Висновок

AI-опис корисний як **швидкий структурний ескіз**, але непридатний
для доказового OSINT без ручної перевірки. Ключова небезпека —
не відверта брехня, а правдоподібні, але непідтверджені зв'язки,
які починають виглядати як факти після читання.

> "Гарно написано" ≠ "правда"
