## ПРОМПТ A — "Поганий" (без ролі, без формату, без обмежень)


### Відповідь Claude (без документів, з тренувальних даних):

> Олексій Чекаль — відомий український каліграф і художник. Він народився в Харкові і здобув освіту в галузі мистецтв. Чекаль відомий своїми роботами в галузі каліграфії та православного мистецтва. Він брав участь у численних виставках в Україні та за кордоном. Чекаль викладає каліграфію і активно пропагує це мистецтво серед широкої аудиторії.

**Проблеми з цією відповіддю:**
- Жодного конкретного факту
- "Відомий" — без вказівки чим саме відомий
- "Численні виставки" — без жодної назви, дати, міста
- "Народився в Харкові" — не підтверджено в джерелах
- Немає організацій, партнерів, дат
- Можлива вигадка кожного твердження

## ПРОМПТ Б — Структурований (роль + формат + обмеження + текст)

📸 **[promt 2.png](promt 2.png)** — ChatGPT: JSON з ПСТГУ, Артос, Russia Cristiana, Rimini Meeting 2013

```
Ти — аналітик OSINT. Витягни з тексту нижче інформацію про актора.

Поля для заповнення:
- повне_імʼя
- поточна_посада
- організація
- освіта
- партнерські_організації (масив)
- проєкти (масив: назва, рік, місце, партнер)
- географія_діяльності (масив)
- публічні_контакти

Правила:
1. Лише факти явно підтверджені в тексті
2. Якщо поле невідоме — вказати null
3. Не вигадуй деталей
4. Цитати мовою оригіналу

Формат: JSON

Текст: [pravmir.ru/kalligrafiya-v-tserkvi/, 2013-01-22]
[Алексей Чекаль — дизайнер, каллиграф, искусствовед. Живет в Харькове.

Преподавал в Харьковской государственной академии дизайна и искусства
и Британской школе дизайна в Москве...]
```

## Порівняння

| Критерій | Промпт A (поганий) | Промпт Б (структурований) |
|---|---|---|
| Конкретні організації | 0 | 7 |
| Конкретні проєкти | 0 | 4 |
| Дати | 0 | 1 |
| Географія | 1 (загальна) | 10 локацій |
| Перевіряємість | ❌ нічого не перевірити | ✅ кожен факт = цитата |
| Ризик галюцинацій | ❌ весь текст | ✅ мінімальний (прив'язано до тексту) |
| Аналітична цінність | Нульова | Production-рівень |

## Висновок

Той самий LLM, та сама людина — але Промпт Б видав розвідувальний продукт,
а Промпт А — розмиту розповідь без єдиного верифікованого факту.

Різниця: роль аналітика + формат JSON + правило "лише з тексту" + конкретний текст-джерело.

**Важливо для OSINT:** структурований промпт без тексту-джерела — все одно ризик галюцинацій.
Захист від них = RAG (прив'язка до конкретних документів), а не тільки гарний промпт.
