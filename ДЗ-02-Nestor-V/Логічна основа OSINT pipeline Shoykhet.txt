Логічна основа OSINT pipeline Shoykhet
Мета
Перетворити сирий контент з відкритих і потенційно ворожих джерел на верифікований, зважений і темпорально маркований аналітичний висновок — захищений від маніпуляцій, резистентний до накопичення неякісних джерел, відтворюваний у часі, отриманий за мінімальну вартість обчислень.

Системна властивість
OSINT відрізняється від звичайного RAG однією критичною рисою: джерела що досліджуються можуть одночасно атакувати дослідника. Ворожий ресурс здатен вкласти приховану інструкцію прямо в публічний текст. Без захисту вона потрапить в контекст LLM і змінить висновок.

Друга властивість: вартість обробки обмежена. Потужний LLM не може читати кожне джерело напряму — це нераціонально за ціною і за ризиком.

Звідси два принципи що визначають всю архітектуру:


┌─────────────────────────────────────────────────────────┐
│  П1: захист на кожному шарі                             │
│      об'єкт аналізу = потенційне джерело атаки          │
├─────────────────────────────────────────────────────────┤
│  П2: дорогий LLM отримує верифікований мінімум          │
│      важка робота делегується дешевшим інструментам     │
└─────────────────────────────────────────────────────────┘
Блок 1 — Дедублікація
Одне джерело має бути оброблене рівно один раз. Повторна обробка того ж матеріалу штучно збільшує кількість «підтверджень» і спотворює статистику гіпотез.


Вхідний потік URL / текстів
        ↓
┌───────────────────────┐
│  URL Gate             │  URL вже є в index.csv?
│  (DEDUP GATE 1)       │──── так ──► ✕ відхилено
└───────────┬───────────┘
            ↓ новий URL
      Jina Reader
      URL → текст
            ↓
┌───────────────────────┐
│  Hash Gate            │  hash(text[:500]) вже є?
│  (DEDUP GATE 2)       │──── так ──► ✕ відхилено (дзеркало)
└───────────┬───────────┘
            ↓ унікальний контент
        далі в pipeline
Два рівні тому що різні URL можуть містити ідентичний текст — дзеркала, репости, синдикований контент.

Блок 2 — Захист від маніпуляцій
Три незалежних точки сканування. Кожна захищає свій рівень, бо атака може прийти з різних напрямків.


[Аналітик] ──► input guard   ──► захист запиту від ін'єкцій
                                  PromptInjection / TokenLimit / Secrets
                    ↓
[Jina текст] ──► ingest guard ──► захист бази від отруєного контенту
                                   PromptInjection / InvisibleText / Secrets
                    ↓
[NotebookLM] ──► chunk guard  ──► захист контексту LLM
                                   сканує SUMMARY, не сирі чанки
                    ↓
[LLM відповідь] ► output guard ──► перевірка релевантності і чутливості
                                    Relevance ≥0.5 / Sensitive redact
Ключовий момент chunk guard: він сканує вихід NotebookLM (~500 токенів), а не оригінальні чанки. Claude ніколи не бачить сирий текст ворожих джерел.

Блок 3 — Довіра до джерела
Не всі джерела рівнозначні. Система присвоює кожному числову стелю довіри що неможливо перевищити незалежно від кількості збігів.


Tier 1-2  незалежні верифіковані ─────────────► cap: 10
Tier 3    регіональні / академічні ───────────► cap: 8
Tier 4-5  афілійовані / слабкі ───────────────► cap: 6 / 4
Tier 6    пропагандистські / РПЦ-медіа ───────► cap: 3
                                                        │
                         confidence > cap? ─────────────┘
                                  ↓
                         confidence = cap
                         status = ГІПОТЕЗА

Cluster A (РПЦ-мережа):
  N джерел ──► підтверджують НАРАТИВ мережі
           ──► не є незалежним підтвердженням факту
           ──► примітка [Cluster A] додається до запису
Блок 4 — Статус висновку
Правила перетворення джерел на верифікований статус. Система резистентна до накопичення: більше слабких джерел не підвищує статус.


Джерела гіпотези
        ↓
┌───────────────────────────────────────────────────────────┐
│  2+ незалежних FACT, Tier ≤3          ──► ПІДТВЕРДЖЕНО    │
│  1 підтверджений FACT, Tier 3         ──► ПЕРЕВІРЯЄТЬСЯ   │
│  будь-яка к-сть FACT, Tier 4+        ──► ГІПОТЕЗА        │
│  будь-яка к-сть, Cluster A           ──► ГІПОТЕЗА        │
└───────────────────────────────────────────────────────────┘
        ↓
  50 Cluster A джерел = ГІПОТЕЗА (cap=3)
  1 незалежне Tier 2 + 1 незалежне Tier 3 = ПІДТВЕРДЖЕНО
Блок 5 — Темпоральна логіка
Та сама дія до і після переломної події — аналітично різні факти. Контекст присвоюється автоматично і впливає на інтерпретацію.


дата публікації / події
        ↓
┌────────────────────────────────────────────────┐
│  до 2014-02-20      ──► "до 2014"              │
│  2014-02-20..       ──► "після Майдану"        │
│  2022-02-24..       ──► "після 24.02.2022"     │
│  невідома           ──► "невідомо"             │
└────────────────────────────────────────────────┘
        ↓
  Мовчання після 24.02.2022 = самостійний сигнал
  (відсутність розриву з мережею — аналітичний факт)
Блок 6 — Вагова логіка
Числова оцінка кожного згадування дозволяє сортування і порівняння через SQL без участі LLM.


W = weight_base × context_mult × mention_type_mult

weight_base     ← афіліація і країна джерела (з registry)
                   RU держ. медіа: 5.0 / RU церковні: 4.0
                   UA національні: 3.5 / INT: 2.5

context_mult    ← характер згадування
                   positive: 1.0 / neutral: 0.7 / critical: -0.5

mention_type    ← тип матеріалу
                   interview: 1.3 / news: 1.0 / list_entry: 0.5
        ↓
  W зберігається в index.csv
  MindsDB: ORDER BY weight DESC — без LLM-виклику
Блок 7 — Прогалини і суперечності
Система фіксує не тільки що підтверджено, але й що невідомо і де є протиріччя.


Нова інформація
        ↓
┌────────────────────────────────────────────────────────┐
│  Підтверджує гіпотезу?   ──► FACT запис                │
│  Суперечить декларації?  ──► DISCREPANCY запис         │
│  Питання без відповіді?  ──► GAP запис                 │
│  Відсутність даних?      ──► GAP:SILENCE (сигнал)      │
└────────────────────────────────────────────────────────┘

  GAP + новий FACT на тій же гіпотезі
        ↓
  сигнал: GAP closeable → closed_by = FACT-ID
Блок 8 — Збереження джерела
Публічні джерела зникають. Ворожі ресурси можуть видалити матеріал якщо відчують увагу. Відтворюваність висновку вимагає архіву.


Нове джерело
        ↓
  archive_priority ← автоматично з tier + cluster
        ↓
┌──────────────────────────────────────────────────────┐
│  forced   Cluster A/B  ──► wayback + локальний знімок│
│  check    Tier 4+ незал ──► wayback або локальний    │
│  optional академічні   ──► тільки wayback            │
└──────────────────────────────────────────────────────┘
        ↓
  archive_url і snapshot_local_path → index.csv
  Алерт якщо forced і архів відсутній
Блок 9 — Вивід і замикання циклу
Кожен новий запис не просто зберігається — він перевіряє стан всієї бази і сигналізує про зміни.


Новий запис збережено
        ↓
┌─────────────────────────────────────────────────────────┐
│  check_loop_closure()                                   │
│                                                         │
│  новий FACT + відкритий GAP (та ж гіпотеза)            │
│       ──► сигнал: "GAP-XXX може бути закритий"          │
│                                                         │
│  новий FACT ПІДТВЕРДЖЕНО + DISC (та ж гіпотеза)        │
│       ──► алерт: "DISC-XXX суперечить FACT-XXX"         │
│                                                         │
│  2+ незалежних FACT Tier≤3 на гіпотезі                 │
│       ──► сигнал: "H-XX — перевір статус гіпотези"      │
└─────────────────────────────────────────────────────────┘
        ↓
  Аналітик отримує:
  [ID] домен | Tier N | Cluster | confidence N | статус
  → --last ID  для повного запису
  → generate_report.py  для звіту по актору
Зведена схема

        [ Аналітик ]
        /           \
  ініціює збір    ставить питання
        ↓                ↓
┌───────────────┐  ┌──────────────────────────┐
│  ПІДГОТОВКА   │  │  АНАЛІЗ + ВИВІД          │

│ Dedup 1 (URL) │  │ NotebookLM → summary     │

                   │ mention-index.csv        │

                   └──────────────────────────┘
                              ↓
                        [ Аналітик ]
